  • showAlertMessage(_:inController:withTitle:block:)

    This method show a standard alert message to inform something to the user.

    The user won’t have any decision but to accept the alert. The alert will only have an ‘ok’ option

    Declaration

    Swift

    public class func showAlertMessage(message : String, inController : UIViewController, withTitle : String, block : (()->())?)

    Parameters

    message

    The body of the alert
    inController

    The controller that is going to present the alert
    withTitle

    The alert view controller title
    block

    An optional code block that is going to be called when the alert is dismissed
  • presentConfirmationAlertViewController(_:description:confirmText:cancelText:controller:destructive:confirmAction:cancelAction:)

    This method is a standard way to confirm potentially dangerous actions to the user. This method is going to be responsable of instanciating and presenting the alert on the view controller.

    Also, this method prepares the alert to tell the calling controller the decision that the user took.

    Declaration

    Swift

    public class func presentConfirmationAlertViewController(title : String, description : String, confirmText : String, cancelText : String, controller : UIViewController, destructive : Bool, confirmAction : ()->(), cancelAction : (()->())?)

    Parameters

    title

    A brief string describing what the user is about to do
    description

    The body of the alert. Tipically the consecuences of performing the action
    confirmText

    The text that is going to appear as a confirmation
    cancelText

    The abort action text
    controller

    The controller that is going to present the alert
    destructive

    If this parameter is set to the confirm action is going to appear in red letters to higthligth that its a dangerous action.
    confirmAction

    The code block that should be called when the user accepts the action
    cancelAction

    An optional code block that should perform some clean up actions when the user cancels its decision such as restoring the user interface or toggling switches to the previous state
